# Security Policy

TSAPS is a cryptographic transaction protocol.

We take security seriously.

## Reporting a Vulnerability

If you discover a security issue:

DO NOT open a public GitHub issue.

Instead, email:

[INSERT SECURITY CONTACT EMAIL]

Include:

- Description of vulnerability
- Steps to reproduce
- Proof-of-concept (if possible)
- Affected versions

We aim to respond within 5 business days.

## Responsible Disclosure

We request a 90-day responsible disclosure window before
public disclosure of critical vulnerabilities.

## Scope

Security issues include:

- Replay vulnerabilities
- Signature bypass
- Idempotency bypass
- Guard bypass
- TLS binding weaknesses
- Formal model inconsistencies

Infrastructure or deployment misconfiguration
issues are out of scope.
